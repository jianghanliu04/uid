## 第二阶段：补齐数据模型

- [ ] 定义地图数据
  - 图片尺寸：`803 x 1404`
  - 比例：`1 pixel = 1.7 meters`
  - 原点：地图左下角
- [ ] 定义坐标转换函数
  - 图像坐标 -> 真实世界坐标
  - 真实世界坐标 -> 图像坐标
- [ ] 定义 IV 数据结构
  - 编号
  - 中心点位置
  - 朝向角度
  - 长宽：`8m x 3m`
  - 可视化缩放因子
- [ ] 定义全局 UI 状态
  - 当前地图
  - 当前旋转角度
  - 已加载 IV 列表
  - 当前交互模式

目标：
先把“地图、坐标、车辆、界面状态”这四类数据固定下来，后面功能才不会乱。

## 第三阶段：完成地图点击与坐标显示

- [ ] 给地图绑定鼠标点击事件
- [ ] 点击地图任意点时读取像素位置
- [ ] 换算成真实世界坐标
- [ ] 在状态栏或文本框里显示坐标

目标：
这是最基础的交互入口，后续距离测量、加载 IV、轨迹测量都依赖点击事件。

## 第四阶段：完成道路有效性判断

- [ ] 设计“道路区域”和“非道路区域”的区分方法
- [ ] 做一个函数判断某个点击点是否位于道路上
- [ ] 非法点返回错误提示
- [ ] 合法点允许继续加载 IV

建议：
先做一个简单可控的方法，例如基于颜色阈值或人工规则判断，再看效果是否足够。

## 第五阶段：完成 IV 加载与显示

- [ ] 提供“添加 IV”按钮

## 第六阶段：完成 IV 删除与位置报告

## 第八阶段：完成地图旋转

- [ ] 提供旋转角度输入控件

## 第九阶段：完成可选功能

- [ ] OR1 道路骨架提取
